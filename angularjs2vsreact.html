Angular JS 2 ou React ?

A melhor escolha JS 2017- Frameworks full-stack

A ReactJS rápida Refresher

Se você vai se lembrar, ReactJS é uma biblioteca JavaScript web criado pelo Facebook, com o objetivo principal de construir interfaces de usuário de elevada performance e responsivos. Reagir foi construído em torno de um dos problemas comuns com outros frameworks JavaScript, ou seja, que trabalham com grandes conjuntos de dados. Ao fazer uso de um DOM virtual, e, aproveitando um método de correção para renderizar novamente somente as partes sujas do DOM quando ocorre uma alteração, Reagir foi capaz de alcançar melhorias de velocidade impressionantes sobre frameworks como Angular 1.x, que tratou o DOM de uma forma mais holística.

Melhorias na Angular 2

Angular 2 adiciona muitas melhorias para as inovações que foram introduzidas no Angular 1. Em primeiro lugar, angular 2 tem um foco muito maior sobre a criação de componentes de front-end reutilizáveis. Enquanto era possível fazê-lo utilizando angular 1, versão 2 do quadro remove muitas das barreiras à reutilização, como a dependência de $ âmbito e nos controladores. Directivas também foram muito simplificada, tornando Angular 2 código muito mais conciso e legível quando comparado com 1.x. Angular Angular 2 também se destina a ser usado com texto datilografado, removendo um monte de o código de validação necessária para impor a segurança de tipos. Juntamente com muitas melhorias de desempenho e de enquadramento, Angular 2 consegue apresentar quase uma aparência totalmente nova para o padrão quadro front-end declarativa.

Algumas advertências

É importante notar a partir do get-go que, em muitos aspectos, comparando Reagir a Angular (qualquer versão) é comparar maçãs com laranjas. Angular é concebido como um quadro front-end, fornecendo uma arquitetura completa para o lado do cliente do seu aplicativo e permitindo que você tratar o seu código de cliente como um conjunto robusto de funcionalidades. Reagir é uma biblioteca, e muito menos intrusivo em termos dos recursos oferecidos - é destinado a ser utilizado como parte de um todo, em vez de ditar a estrutura do seu código. Embora isso possa parecer para fazer uma comparação entre os dois um pouco difícil de realizar, há semelhanças suficientes entre os dois para conduzir uma discussão frutífera sobre as vantagens e desvantagens de cada um.

Arquitetura

Como mencionado acima, Reagir é uma biblioteca JavaScript usado para criar modelos de resposta que podem ser rapidamente atualizados e prestados ao usuário. Como tal, não há nenhum foco de arquitetura real na Reagir - é sim uma ferramenta para ser usada dentro do contexto maior de código do lado do cliente para uma aplicação. Angular 2, por outro lado, baseia-se nas opiniões introduzidas no angular 1, e, como resultado cristaliza a visão do quadro do que o "caminho certo" de desenvolvimento de código front-end deve ser parecida. Com o foco em componentes e tipo de segurança, Angular 2 permite que você se concentre em modularizing seu código, a construção de um front-end orientada a objetos com foco na reutilização e encapsulamento. A partir desta perspectiva, reagir e angular 2 realmente não comparar muito fortemente - Reagir é a escolha certa se você está procurando mais flexibilidade na arquitetura de seu código, onde, com angular 2 que você está escolhendo a adotar essencialmente uma arquitetura para frente do seu aplicativo -END como um todo. Qual padrão é "melhor" é, em última análise, então, uma questão de que se encaixa padrões de design do seu aplicativo melhor.

Conclusão

A escolha entre Angular 2 e reagir se resume a uma preferência de estilo. Reagir, como uma biblioteca focada na velocidade de renderização, é uma ferramenta útil para lidar com apresentações UI grandes e complexas em seu cliente ou aplicativos nativos. Angular 2, por outro lado, tem uma visão muito mais ampla do processo de desenvolvimento. Como um quadro opinativo, ele está olhando para orientar a maneira em que você construir seus aplicativos, ao mesmo tempo, permitindo-lhe criar experiências de UI expressivas e reutilizáveis. Em última análise, a decisão irá variar de acordo com os objetivos da sua aplicação, bem como as opiniões de sua equipe de desenvolvimento. Se você favorece o desempenho de grandes interfaces complexas, Reagir é uma boa escolha, enquanto que, se você quer se concentrar em melhorar o processo de desenvolvimento e arquitetura de seu código, Angular 2 será o caminho certo a seguir. Como sempre, por entender o seu espaço de problema e as necessidades e desejos de sua equipe de desenvolvimento, você acabará por não ser capaz de tomar uma decisão errada.
